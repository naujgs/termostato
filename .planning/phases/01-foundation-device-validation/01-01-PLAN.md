  <action>
1. Create the file CoreWatch/CoreWatch/CoreWatch-Bridging-Header.h with this exact content:

```objc
#ifndef CoreWatch_Bridging_Header_h
#define CoreWatch_Bridging_Header_h

#import <Foundation/Foundation.h>

// IOKit — used by Phase 1 IOKit probe only.
// This header will remain after Phase 1; the probe code in ViewModel is removed.
extern CFTypeRef IOServiceGetMatchingService(mach_port_t mainPort, CFDictionaryRef matching);
extern CFMutableDictionaryRef IOServiceMatching(const char *name);
extern kern_return_t IORegistryEntryCreateCFProperties(
    io_object_t entry,
    CFMutableDictionaryRef *properties,
    CFAllocatorRef allocator,
    IOOptionBits options
);
extern kern_return_t IOObjectRelease(io_object_t obj);

#endif /* CoreWatch_Bridging_Header_h */
```

2. Wire the bridging header to the project:
   - In Xcode: select the CoreWatch target → Build Settings → search "Objective-C Bridging Header"
   - Set the value to: CoreWatch/CoreWatch-Bridging-Header.h
   - This sets SWIFT_OBJC_BRIDGING_HEADER in project.pbxproj

3. Add IOKit.framework to the target's Frameworks, Libraries, and Embedded Content:
   - Target → General → Frameworks, Libraries, and Embedded Content → "+" → search "IOKit" → add IOKit.framework
   - Set embed to "Do Not Embed"

4. Build again to confirm the bridging header compiles without errors.

NOTE: These IOKit extern declarations replicate the public-symbol signatures. No IOKit.framework umbrella header is available in the public iOS SDK — the externs are the correct approach for a sideloaded app using the private registry API.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Xcode build system → device | Code signing gate; free Apple ID caps what entitlements can be embedded |
| App process → IOKit registry | Private API call; AMFI enforcement is the boundary — app may receive nil silently |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | project.pbxproj | accept | Single-developer local project, no CI pipeline; no external write access to .xcodeproj |
| T-01-02 | Elevation of Privilege | IOKit bridging header | accept | Externs grant no new entitlements; AMFI still enforces `systemgroup.com.apple.powerlog` requirement at runtime — the probe will return nil if blocked, which is the expected and logged outcome |
| T-01-03 | Denial of Service | Free Apple ID certificate | accept | 7-day cert expiry is a known constraint documented in CLAUDE.md; re-install weekly is the accepted operational model |
| T-01-04 | Information Disclosure | Console log output | accept | `print()` output visible only via Xcode debugger console attached to the developer's own device; no remote logging, no third-party analytics |
</threat_model>
