**1. APIVerdict enum (D-06):**
```swift
enum APIVerdict: String {
    case accessible = "Accessible"   // KERN_SUCCESS + non-zero plausible data
    case degraded   = "Degraded"     // KERN_SUCCESS but zeroed or stale data
    case blocked    = "Blocked"      // KERN_FAILURE or other error code
    case pending    = "Pending"      // Not yet probed
}
```

**2. MachProbeResult struct (D-09):**
```swift
struct MachProbeResult: Identifiable {
    let id = UUID()
    let api: String              // e.g. "host_statistics (CPU)"
    let kernReturn: kern_return_t
    let verdict: APIVerdict
    let rawData: String          // Human-readable dump of values
    let timestamp: Date
}
```

**3. SystemMetricsProbe class:**
- Annotate `@Observable @MainActor final class SystemMetricsProbe`
- Properties:
  - `private(set) var results: [String: [MachProbeResult]] = [:]` — keyed by API name, array of up to 3 samples
  - `private(set) var finalVerdicts: [String: APIVerdict] = [:]` — majority verdict per API after 3 samples
  - `private(set) var isProbing: Bool = false`
  - `private(set) var samplesCompleted: Int = 0`
  - `private var probeTask: Task<Void, Never>?`
- API name constants (match UI-SPEC copywriting contract exactly):

  - `static let memoryAPI = "host_statistics64 (Memory)"`
  - `static let processMemoryAPI = "task_info (Process Memory)"`
  - `static let processCPUAPI = "task_threads (Process CPU)"`
  - `static let allAPIs = [cpuAPI, memoryAPI, processMemoryAPI, processCPUAPI]`

**4. Four probe methods (per D-02, D-03 — pure Swift with withUnsafeMutablePointer):**

`private func probeSystemCPU() -> MachProbeResult`:
- Call `host_statistics(mach_host_self(), HOST_CPU_LOAD_INFO, ...)` using `withUnsafeMutablePointer` + `withMemoryRebound` pattern from RESEARCH.md Pattern 1.
- Count = `MemoryLayout<host_cpu_load_info_data_t>.size / MemoryLayout<integer_t>.size`
- If KERN_SUCCESS: check `cpu_ticks.0 + cpu_ticks.1 + cpu_ticks.2 + cpu_ticks.3 > 0` for accessible vs degraded
- rawData format: `"user: \(info.cpu_ticks.0), system: \(info.cpu_ticks.1), idle: \(info.cpu_ticks.2), nice: \(info.cpu_ticks.3)"`
- Print `[CoreWatch] host_statistics CPU: kern_return_t=\(result), data=\(rawData)`

`private func probeSystemMemory() -> MachProbeResult`:
- Call `host_statistics64(mach_host_self(), HOST_VM_INFO64, ...)` using same pointer pattern.
- Count = `MemoryLayout<vm_statistics64_data_t>.size / MemoryLayout<integer_t>.size`
- If KERN_SUCCESS: check `free_count + active_count + inactive_count + wire_count > 0` for accessible vs degraded
- rawData format: `"free: \(vmStat.free_count), active: \(vmStat.active_count), inactive: \(vmStat.inactive_count), wired: \(vmStat.wire_count)"`
- Print `[CoreWatch] host_statistics64 Memory: kern_return_t=\(result), data=\(rawData)`

`private func probeTaskMemory() -> MachProbeResult`:
- Call `task_info(mach_task_self_, task_flavor_t(MACH_TASK_BASIC_INFO), ...)` using pointer pattern.
- Count = `mach_msg_type_number_t(MACH_TASK_BASIC_INFO_COUNT)`
- If KERN_SUCCESS: check `resident_size > 0` for accessible vs degraded
- rawData format: `"resident_size: \(info.resident_size) bytes (\(info.resident_size / 1024 / 1024) MB), virtual_size: \(info.virtual_size) bytes"`
- Print `[CoreWatch] task_info Memory: kern_return_t=\(result), data=\(rawData)`

`private func probeTaskCPU() -> MachProbeResult`:
- Call `task_threads(mach_task_self_, &threadList, &threadCount)`.
- Enumerate threads, call `thread_info` with `THREAD_BASIC_INFO` flavor for each.
- Sum `cpu_usage / TH_USAGE_SCALE * 100.0` for non-idle threads (check `flags & TH_FLAGS_IDLE == 0`).
- CRITICAL: Deallocate thread list with `vm_deallocate(mach_task_self_, vm_address_t(bitPattern: threads), vm_size_t(MemoryLayout<thread_act_t>.size * Int(threadCount)))` — use defer block.
- rawData format: `"threads: \(threadCount), total_cpu: \(String(format: "%.1f", totalUsage))%"`
- Print `[CoreWatch] task_threads CPU: kern_return_t=\(result), data=\(rawData)`

**5. Probe sequence method (D-07):**

`func runProbeSequence()`:
- Guard `!isProbing` else return
- Set `isProbing = true`, `samplesCompleted = 0`, clear `results` and `finalVerdicts`
- Create `probeTask = Task { ... }` that:
  - Loops 3 times (samples 0, 1, 2):
    - Calls all 4 probe methods, appends each result to `results[apiName]`
    - Increments `samplesCompleted`
    - Prints `[CoreWatch] Probe sample \(i+1) of 3 complete`
    - If not last sample: `try? await Task.sleep(for: .seconds(10))` — check `Task.isCancelled` before continuing
  - After 3 samples: compute `finalVerdicts` — majority verdict across 3 samples per API (if 2+ accessible = accessible, 2+ degraded = degraded, etc.)
  - Set `isProbing = false`
  - Print `[CoreWatch] Probe sequence complete. Verdicts: \(finalVerdicts)`

`func cancelProbe()`:
- `probeTask?.cancel()`
- `isProbing = false`

**Concurrency notes:**
- Use `@MainActor` on the class per established pattern. Mach calls are microsecond-fast kernel traps — safe on main thread (RESEARCH.md A5).
- Use `nonisolated(unsafe)` if any stored Mach types trigger sendability warnings — follow the TemperatureViewModel pattern for `thermalObserver`.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| App process -> Mach kernel | Mach API calls cross from userspace sandbox into kernel; kernel enforces access control |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Information Disclosure | host_statistics / host_statistics64 | accept | System-wide metrics are non-sensitive (CPU ticks, memory page counts). No PII, no credentials. Apple's sandbox is the access control — we probe it, not bypass it. |
| T-06-02 | Denial of Service | task_threads thread enumeration | mitigate | Deallocate thread list with vm_deallocate in defer block after every task_threads call to prevent mach port leak. Probe runs max 3 times per invocation. |
| T-06-03 | Elevation of Privilege | Mach API calls | accept | All calls use public mach_host_self() / mach_task_self_ — no private entitlements requested. If sandbox blocks, we document and fall back gracefully. |
</threat_model>
