<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 1: On-device SC5 tab persistence verification</name>
  <read_first>
    - .planning/phases/08-dashboard-tabs/08-CONTEXT.md — review D-04 for exact SC5 test criteria
  </read_first>
  <what-built>
    Plan 08-01 added `@State private var selectedTab: Int = 0` to ContentView and bound it to `TabView(selection: $selectedTab)` with `.tag(0/1/2)` on each tab. The app must be built and installed on device to test this change.
  </what-built>
  <how-to-verify>
    Install and run the updated build on your physical iOS 18 device (Xcode → Product → Run, or Xcode → Window → Devices and Simulators → Install).

    **SC5 test sequence (per D-04):**

    1. Launch the app. Wait ~5 seconds for the first poll to complete.
    2. Navigate to the CPU tab. Confirm "App CPU" card shows a non-zero percentage (e.g. "4.2%").
    3. Navigate to the Memory tab. Confirm "App Memory" shows a non-zero MB value, "Memory Free" and "Memory Used" show non-zero GB values.
    4. Navigate back to the Thermal tab. Confirm the thermal state badge still shows the current state (Nominal/Fair/Serious/Critical) and the history chart is visible. Confirm the chart has accumulated data points (not reset to empty).
    5. Navigate to CPU tab again. Confirm "App CPU" shows its last polled value (not reset to "—"). Wait for the next poll (~5s) to confirm the value updates.
    6. Navigate to Memory tab and back. Confirm values persist (not "—") between navigation.
    7. On the Thermal tab, long-press the "CoreWatch" title. Confirm the MachProbeDebugView debug sheet opens. Dismiss it. Confirm no regression in Thermal tab after dismissal.

    **Pass criteria:**
    - No tab resets to "—" when returning to it after navigating away
    - Thermal chart does not clear when switching to another tab and back
    - Debug sheet still accessible from Thermal tab
    - No crashes or visible layout regressions

    **Note:** Current views have no ScrollView — scroll position is not a concern per D-04.
  </how-to-verify>
  <acceptance_criteria>
    - All 7 steps above pass with no unexpected behavior observed
    - No metric card flashes back to "—" on tab return
    - Thermal badge and chart are intact after tab switching
    - Debug sheet opens and dismisses cleanly
  </acceptance_criteria>
  <resume-signal>
    Type "approved" if all 7 steps pass. If any step fails, describe what went wrong so the executor can diagnose and fix.
  </resume-signal>
  <done>User confirms SC5 passes on physical device. All tabs display correct data. No regressions.</done>
