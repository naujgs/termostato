<objective>
Add UIApplication.beginBackgroundTask to TemperatureViewModel so the thermalStateDidChangeNotification observer stays live after the app backgrounds.

Purpose: iOS suspends CoreWatch almost immediately after backgrounding because the app has no declared background execution mode. The thermalStateDidChangeNotification observer registered in init() cannot fire on a suspended process, so background thermal alerts never deliver. beginBackgroundTask requests ~30 seconds of execution time — enough for the thermal observer to fire and schedule a local notification.

Output: TemperatureViewModel.swift with a backgroundTaskID stored property, background task requested in stopPolling(), and background task ended in startPolling().
</objective>

```swift
func stopPolling() {
    timerCancellable?.cancel()
    timerCancellable = nil
    // Request background execution time so the thermal observer can fire (iOS ~30s window).
    let task = UIApplication.shared.beginBackgroundTask(withName: "ThermalMonitor") { [weak self] in
        guard let self else { return }
        let id = self.backgroundTaskID
        UIApplication.shared.endBackgroundTask(id)
        self.backgroundTaskID = .invalid
    }
    backgroundTaskID = task
    print("[CoreWatch] Polling stopped. Background task \(task.rawValue) started.")
}
```
