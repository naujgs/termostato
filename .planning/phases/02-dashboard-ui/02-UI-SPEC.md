## Spacing Scale

SwiftUI spacing uses points (1 pt = 1 px on non-retina; visually equivalent to CSS px for layout purposes). All values are multiples of 4.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4pt | Icon gaps, label-to-value gap inside badge |
| sm | 8pt | Tight padding within components |
| md | 16pt | Default `.padding()` — view edges, VStack spacing |
| lg | 24pt | Badge internal vertical padding (top + bottom) |
| xl | 32pt | Vertical gap between badge and chart |
| 2xl | 48pt | Not used in this phase |
| 3xl | 64pt | Not used in this phase |

Exceptions:
- Touch targets: minimum 44pt per Apple HIG — badge height will naturally exceed this via `lg` padding on a display-size font.
- Badge corner radius: 20pt (fixed, not a spacing token — matches iOS card idiom).

Source: Apple HIG minimum touch target; D-01/D-02 from CONTEXT.md

## Typography

All sizes in points. SwiftUI semantic styles map to these values on standard iOS 18 Dynamic Type at default size.

| Role | SwiftUI Style | Approx Size | Weight | Line Height |
|------|--------------|-------------|--------|-------------|
| Badge state label | `.largeTitle` | 34pt | Bold (700) | 1.2 |
| Section header / sub-label | `.headline` | 17pt | Bold (700) | 1.3 |
| Chart axis labels | `.caption` | 12pt | Regular (400) | 1.4 |
| App title / nav | `.title2` | 22pt | Regular (400) | 1.2 |

Weights used: Regular (400) and Bold (700).

Implementation note: apply `.fontWeight(.bold)` override on the `.headline` role — the system default Semibold weight is not used. This keeps the declared weight set to exactly two values.

Rationale:
- Badge uses `.largeTitle` + bold so the state name dominates the screen half at a glance (D-01, CONTEXT specifics: "impossible to misread").
- Section header / sub-label uses `.headline` + `.fontWeight(.bold)` override — size distinction (17pt vs 12pt caption) plus consistent bold weight creates hierarchy without introducing a third weight class.
- Chart axis labels use `.caption` because chart density at 120 data points demands minimal label footprint.

Source: D-01 (badge glanceability), Claude's Discretion (exact typography), Apple HIG Dynamic Type scale

## Color

### Semantic Colors (System Appearance — Light + Dark)

App follows system appearance (D-07 from CONTEXT.md). The four thermal-state colors are the primary accent palette; they replace a single accent color with a four-way semantic palette.

| Role | Light Mode Value | Dark Mode Value | Usage |
|------|-----------------|-----------------|-------|
| Dominant (60%) | System Background (`Color(.systemBackground)`) | System Background | Root view background, chart background |
| Secondary (30%) | Secondary System Background (`Color(.secondarySystemBackground)`) | Secondary System Background | Badge container fill when state is Nominal — provides contrast floor |
| Nominal | `Color.green` (system) | `Color.green` (system) | Badge fill + chart step line when state = Nominal |
| Fair | `Color.yellow` (system) | `Color.yellow` (system) | Badge fill + chart step line when state = Fair |
| Serious | `Color.orange` (system) | `Color.orange` (system) | Badge fill + chart step line when state = Serious |
| Critical | `Color.red` (system) | `Color.red` (system) | Badge fill + chart step line when state = Critical |
| Destructive | n/a | n/a | No destructive actions in this phase |

Accent reserved for: badge background fill (thermal-state-mapped color) and chart step series color. No other elements use the thermal-state colors.

Badge label text color:
- Nominal + Fair states → `.primary` (dark text on light fill)
- Serious + Critical states → `.white` (light text on saturated fill)
- Use `foregroundStyle(.primary)` with `.colorScheme` override, or hardcode per state.

Source: D-07 (system appearance), D-08 (four-color thermal mapping), REQUIREMENTS.md DISP-01

## Layout Contract

This section is required because this is a single-screen dashboard with explicit spatial rules.

### Screen Structure

```
┌──────────────────────────────────┐
│  [App title — small, top]        │  title area: .title2, padding md
├──────────────────────────────────┤
│                                  │
│   ┌──────────────────────────┐   │
│   │                          │   │  badge: full-width rounded rect
│   │      NOMINAL             │   │  corner radius 20pt
│   │                          │   │  internal padding: lg (24pt) top/bottom
│   └──────────────────────────┘   │  font: .largeTitle bold
│                                  │
│   xl gap (32pt)                  │
│                                  │
│   ┌──────────────────────────┐   │
│   │   [Step Chart]           │   │  chart: fills remaining height
│   │   120 data points        │   │  Swift Charts — LineMark + interpolation .stepEnd
│   │   Y: state levels        │   │
│   │   X: time (session)      │   │
│   └──────────────────────────┘   │
└──────────────────────────────────┘
```

### Badge

- Full-width `RoundedRectangle(cornerRadius: 20)` with `.fill(thermalStateColor)`.
- State name in `.largeTitle` bold, centered, `foregroundStyle` white or primary based on state (see Color section).
- No border — color fill provides sufficient contrast.
- Not tappable in Phase 2 (no action needed).

### Chart

- Swift Charts `Chart` view.
- Data series: `LineMark` with `interpolationMethod: .stepEnd` — produces the step-chart behavior required by D-03.
- Y axis: discrete `Int` mapping (Nominal=0, Fair=1, Serious=2, Critical=3).
- Y axis labels: show state names — "Nominal", "Fair", "Serious", "Critical" — at their respective Y positions. (Claude's Discretion resolved: state names on Y-axis are more readable than color bands alone given the step-chart format.)
- X axis: omit explicit time labels. At 120 readings (60 min window), per-sample labels would collide. Use `.hidden()` on `AxisMarks` for X. A secondary text label below the chart reads "Session history (last 60 min)" in `.caption` `.secondary`.
- Chart series color: dynamic — each data point rendered in its thermal-state color. Achieved via `foregroundStyle` mapped from the Y value.
- Chart height: `.frame(minHeight: 200)` — fills remaining space after badge with a minimum floor.
- Animation: `.animation(.easeInOut(duration: 0.3), value: history)` on the `Chart` view — smooth append animation without jarring jumps.
- Chart does NOT scroll (D-04). It always displays the full session window within bounds.

Source: D-03 (step chart), D-04 (no scroll), D-05 (120-reading ring buffer), Claude's Discretion (Y-axis labels, X-axis omission, animation)

| Element | Copy |

| Primary CTA | None — this phase has no user-initiated actions (display only) |

| Error state | Not applicable — `ProcessInfo.thermalState` never throws; no error state needed in Phase 2 |
| Destructive confirmation | None — no destructive actions in this phase |

Empty state context: On cold launch, history array is empty. The chart area shows the empty-state copy until the first poll fires (immediate on `startPolling()`). In practice this is sub-second; the copy is a safety net.

Source: REQUIREMENTS.md DISP-01/DISP-02; D-06 (session-only, starts fresh); Claude defaults for copy

## Interaction States

| Component | State | Behavior |
|-----------|-------|----------|
| Badge | Nominal | Green fill, dark text, corner radius 20pt |
| Badge | Fair | Yellow fill, dark text |
| Badge | Serious | Orange fill, white text |
| Badge | Critical | Red fill, white text |
| Chart | Empty (0 readings) | Show empty-state copy, no chart rendered |
| Chart | Filling (1–119 readings) | Renders all available points; left edge starts at app-open time |
| Chart | Full (120 readings) | Ring buffer evicts oldest; chart pans left continuously |
| App | Foregrounded | Polling active, badge + chart updating every 30s |
| App | Backgrounded | Polling paused (scenePhase .background); badge frozen at last state |
